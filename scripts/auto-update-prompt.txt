IPV6_FRAG_ESCAPE is an unprivileged container-to-host-root escape in the
Linux kernel: an arithmetic error in the IPv6 fragmentation path
(__ip6_append_data() in net/ipv6/ip6_output.c, IPv4 sibling
__ip_append_data() in net/ipv4/ip_output.c) undersizes an allocation so the
packet head overflows into skb_shared_info, giving control of nr_frags — a
page use-after-free the PoC turns into a Dirty-Pagetable root shell. The
canonical fix is 736b380e28d0 (IPv6) plus eca856950f7c (IPv4), merged as
38becddc and first released in v7.2-rc1. The bad accounting was introduced
dormant in v6.0 (773ba4fe9104 / 8eb77cc73977) and armed in v6.6 by
ce650a166335. The reachable window is kernels 6.6 through 7.1. There is NO
CVE.
