# Website plan — `https://kimmo.cloud/ipv6_frag_escape/`

Publish the IPV6_FRAG_ESCAPE tracker as a single-page static site,
generated by **Hugo** locally and rsync'd to a personal nginx server
(`haig`).  The build infrastructure mirrors the sibling cifswitch /
pintheft / CVE-2026-43284 trackers; only the content and a few config
strings differ.

## Architecture

- **Source repo:** `github.com/suominen/ipv6_frag_escape` (this repo).
- **Generator:** Hugo extended, ≥ 0.146.0.  Run locally; nothing built
  in CI.
- **Source layout:** Hugo project under `site/`.  The tracker is a
  single page at `site/content/_index.md`.
- **Theme:** PaperMod, integrated as a Hugo Module (no submodule).  See
  `site/hugo.toml` and `site/go.mod`.
- **Dev environment:** Nix flake (`flake.nix`) provides hugo + go + git
  + resvg (for `make banner`) + curl/libarchive/zstd (the auto-update
  agent's RPM-unpacking tools).  Auto-activates via `.envrc` if direnv
  is installed.
- **Build:** `make build` → `hugo --minify --gc --cleanDestinationDir`,
  output in `site/public/`.
- **Publish:** `make dist` → `rsync -avz --delete site/public/`
  → `haig:/ipv6_frag_escape/`.
- **Web server:** existing nginx vhost on `kimmo.cloud` serves
  `htdocs/ipv6_frag_escape/` directly at the URL path `/ipv6_frag_escape/`.

## Naming — nickname slug (no CVE)

The bug has **no CVE** and no marketable/media name — the Finnish NCSC
advisory and AlmaLinux both call it `IPV6_FRAG_ESCAPE`; TuxCare writes it
`ipv6-frag-escape`.  The site therefore uses the **PoC repo's name** as its
slug, with underscores, everywhere:

- repo: `~/src/ipv6_frag_escape`, `github.com/suominen/ipv6_frag_escape`
- Go module path: `github.com/suominen/ipv6_frag_escape/site`
- URL: `https://kimmo.cloud/ipv6_frag_escape/`
- systemd units: `ipv6-frag-escape-tracker-update.{service,timer}`

If a CVE is later assigned, the decision (per sibling precedent, which runs
both ways) is to **stay on the nickname slug** unless the user chooses to
retarget; the CVE would then live in the tracker body, Summary, and
references only.  No redirect is needed since the slug never moved.

## Tracker content

- One kernel bug, **single verdict axis**: the `736b380e28d0` /
  `eca856950f7c` fix (merged as `38becddc`, v7.2-rc1) is the only thing
  that flips a row.  The IPv6/IPv4 fragmentation path is always built in,
  so there is **no userspace reachability gate** (unlike cifswitch's
  cifs-utils second axis).  `CONFIG_INIT_ON_ALLOC_DEFAULT_ON` is a
  root-vs-DoS *discriminator* recorded in prose, not a column.
- **Distribution status** uses a single **combined table** placed
  immediately under the heading (`Distribution | Release | Kernel |
  Fixed since | Status`), with per-distro `###` headings retained only
  where there is an audience-relevant note (Debian/PVE DoS-only rationale;
  NixOS forced `init_on_alloc`; EL10-family root case with AlmaLinux as the
  leading indicator).  This follows the combined-table + sticky "Fixed
  since" layout of the cifswitch tracker.
- Initial rows are **seeded with real, sourced status** (2026-07-03): the
  reachable window is kernels 6.6–7.1, so distros outside it are *not
  affected*; in-window distros are `:x:` (EL10, root) or `:warning:`
  (Debian/PVE/NixOS, DoS-only).  No unconfirmed "fixed" claims.
- Social/OpenGraph banner — `site/assets/ipv6-frag-escape-tracker.svg`
  rasterised to `site/static/ipv6-frag-escape-tracker.png` by `make banner`
  (resvg), wired into the `cover:` front-matter.

## Decisions

- **Hosting:** own nginx on `haig`, *not* GitHub Pages.  Same as the
  sibling trackers.
- **URL:** `https://kimmo.cloud/ipv6_frag_escape/` (nickname slug; see
  "Naming").
- **Theme integration:** Hugo Modules; theme PaperMod.
- **Canonical source:** `site/content/_index.md`.
- **Automated maintenance:** a user-level systemd timer
  (`systemd/ipv6-frag-escape-tracker-update.timer`, twice daily) runs
  `scripts/auto-update`, which merges `origin/main` into a dedicated
  long-lived `auto-update` branch in a separate worktree and hands off to
  headless Claude with `scripts/auto-update-prompt.txt`.  The agent only
  commits onto `auto-update` — it does not push or open PRs.  Merges of
  `auto-update` into `main` are done manually.  The wrapper prints an
  `IPV6_FRAG_ESCAPE tracker auto-update starting …` banner as its first
  output line so syslog summaries that aggregate several trackers can be
  told apart (all otherwise log under the generic `auto-update` process
  name).

## Risks & gotchas

- **Anchor drift.** Hugo/Goldmark's heading slugger differs from
  GitHub's for headings with em dashes, parens, plus signs, or slashes.
  Pin brittle headings with `### Heading {#stable-id}` if external
  bookmarks need to survive renames.
- **Subpath baseURL.** `baseURL = "https://kimmo.cloud/ipv6_frag_escape/"`
  must include the trailing slash and the path component.
- **`.cleanDestinationDir`** wipes `site/public/` before each build.
- **rsync `--delete`** removes server-side files not present in the build
  output.  Don't store unrelated content under `htdocs/ipv6_frag_escape/`.
- **Client-side table tweaks.** `layouts/partials/extend_footer.html`
  rewrites tables in the browser: it collapses consecutive duplicate
  first-column cells via `rowSpan` (which keeps the combined distro
  table's "Distribution" column clean), and tags cells whose text starts
  with a status emoji with `class="status"` so `custom.css` can
  hang-indent them.  With JavaScript disabled both tweaks are skipped —
  the table still renders correctly, just un-compacted.

## Social banner

The OpenGraph / social-preview image is generated from an SVG source:
`site/assets/ipv6-frag-escape-tracker.svg` (1200×630) is rasterised to
`site/static/ipv6-frag-escape-tracker.png` by `make banner`, which runs
`resvg`.  The PNG is committed so ordinary `make build` / `make dist` runs
need no rasteriser.  `resvg` needs the Roboto fonts (`fonts-roboto-unhinted`
on Debian) — without them it silently drops the SVG's sans-serif text.
